# 封装

### 包的概念

---

- 包类似文件夹结构，为了开发方便，我们一般将功能相似的类归为一个包
- 包的命名规范一般是:公司域名.公司名.项目名.包名
- 包的定义:

```java
package com.skyme.demo.test
```

### 访问修饰符

---

| 修饰符 | 同一个类 | 同一个包 | 子类 | 任何地方 |
| --- | --- | --- | --- | --- |
| public(公共的) | Y | Y | Y | Y |
| protected(受保护的) | Y | Y | Y |  |
| default（默认的） | Y | Y |  |  |
| private（私有的） | Y |  |  |  |

### 封装

---

类似于我们将电脑的功能封装在机箱里面这种

#### 封装的优点:

- 良好封装减少耦合
- 类内部结构可以自由修改
- 对成员变量精确的控制
- 隐藏信息，实现细节

#### 封装的实现：

```java
public class Person{
    private String name;
    private int age;
​
    public int getAge(){
      return age;
    }
​
    public String getName(){
      return name;
    }
​
    public void setAge(int age){
      this.age = age;
    }
​
    public void setName(String name){
      this.name = name;
    }
}
```

为了保护成员变量我们将成员变量设置为private，我们设置了setter和getter方法

采用this解决局部变量和成员变量直接的名字冲突
